# micro_site

## Scope

HTML multi-page microsite for launches, campaigns, product education, events, or focused brand storytelling.

## Standard Pages

1. Home or overview
2. Product, offer, or story detail
3. Proof, use cases, resources, or comparison
4. Pricing, contact, signup, or next step
5. About, FAQ, or support when needed

## Required Inputs

- Site goal
- Audience and intent stage
- Requested page list when available
- Primary conversion path
- Brand, product, image, or content references

## Design System Hints

- Keep navigation, footer, CTA language, and repeated components consistent across pages.
- Make each page self-contained, but keep the site path obvious.
- Prefer local `.html` links that match planned page paths.
- Use page-scoped section ids when different pages repeat similar section names.

## Variation Axes

- Site map depth
- Navigation density
- Story-led versus task-led page order
- Proof and conversion emphasis
- Visual rhythm across repeated page chrome

## Review Fields

- Page list and paths
- Cross-page navigation
- Primary conversion path
- Repeated header/footer treatment
- Required assets per page

## QC Focus

- Every planned page has a matching HTML artifact.
- Navigation links align with planned local `.html` paths.
- Header, footer, CTA, and repeated components remain visually consistent.
- Page-specific content is not duplicated blindly across the site.
- HTML and asset paths remain portable.
