# one_pager

## Scope

Single HTML page that explains one product, service, initiative, profile, proposal, or event in a compact scroll.

## Standard Sections

1. Opening statement or offer
2. Context and audience need
3. Core message, benefits, or argument
4. Details, proof, or supporting modules
5. Timeline, process, specs, or package summary
6. Final action or contact path

## Required Inputs

- Topic or offer
- Audience
- Desired reader action
- Essential content points
- Tone and density constraints

## Design System Hints

- Keep the page concise and avoid microsite-style navigation unless anchors help scanning.
- Use clear section transitions and a strong reading path.
- Prefer dense, well-structured modules over oversized marketing composition.
- Keep the first viewport focused on the exact topic, not generic positioning.

## Variation Axes

- Editorial versus operational tone
- Content density
- Anchor navigation
- Proof depth
- Final action prominence

## Review Fields

- Topic clarity
- Content order
- Reader action
- Density
- Supporting proof

## QC Focus

- The page can be understood as one continuous artifact.
- The first viewport states the concrete topic or offer.
- Section order supports the intended reading path.
- Mobile layout remains readable without horizontal overflow.
- HTML remains portable without local absolute paths.
