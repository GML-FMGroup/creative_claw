# wireframe

## Scope

HTML wireframe for early information architecture, layout planning, workflow structure, or page skeleton review.

## Standard Sections

1. Frame, header, or app/page shell
2. Primary content region
3. Secondary panel, aside, or supporting content
4. Controls, forms, table, or list structure
5. Empty, loading, or placeholder modules
6. Notes or handoff annotations when useful

## Required Inputs

- Target page or screen
- Main user goal
- Required content blocks
- Priority order
- Fidelity constraints

## Design System Hints

- Use neutral grayscale structure and restrained accents.
- Emphasize hierarchy, spacing, grouping, and scan path over final visual polish.
- Label regions clearly enough for review, but avoid long explanatory text.
- Keep dimensions stable for reusable modules and responsive breakpoints.

## Variation Axes

- Fidelity level
- Layout density
- Navigation placement
- Content priority
- Annotation depth

## Review Fields

- Content blocks
- Priority order
- Layout hierarchy
- Responsive structure
- Missing states

## QC Focus

- Wireframe reads as structure, not a finished marketing page.
- Main content hierarchy is clear at desktop and mobile sizes.
- Placeholder content does not pretend to be final copy.
- Repeated regions use consistent spacing and alignment.
- HTML remains portable and easy to revise.
