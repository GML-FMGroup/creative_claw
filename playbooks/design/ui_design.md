# ui_design

## Scope

HTML UI mockup for dashboards, tools, SaaS workspaces, admin panels, and app screens.

## Standard Sections

1. Navigation or app frame
2. Primary workflow controls
3. KPI or summary row
4. Main work surface
5. Detail panel, table, chart, or task list
6. Empty, warning, or next-action state

## Required Inputs

- Target user role
- Main workflow
- Data or object types shown on screen
- Density preference
- Device targets

## Design System Hints

- Favor quiet, work-focused visual design.
- Avoid marketing-style hero sections for operational tools.
- Keep information hierarchy compact and scannable.
- Use predictable controls for filters, tables, tabs, and actions.

## Variation Axes

- Density
- Navigation placement
- Table versus card emphasis
- Metric hierarchy
- Alert prominence

## Review Fields

- User role
- Workflow priority
- Information density
- Data modules
- Responsive behavior

## QC Focus

- Controls are easy to scan and repeat.
- Compact headings do not use hero-scale typography.
- Mobile layout remains usable without horizontal overflow.
- Important status and action states are represented.
- HTML contains stable section ids for future local revisions.
